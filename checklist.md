### 1. Подготовка инфраструктуры (Фундамент)
*   [ ] **Развертывание Neo4j:** Поднять инстанс (можно локально или в облаке), развернуть схему узлов по вашей Miro-карте.
*   [ ] **Backend-скелет (Кирилл):** Настроить FastAPI, который умеет принимать файлы и сохранять их в `Object Storage` (MinIO/S3).
*   [ ] **Настройка API:** Получить/проверить доступ к OpenAI API.

### 2. Пайплайн обработки данных (Интеграция)
*   [ ] **Парсер:** Реализовать цепочку: PDF $\rightarrow$ OCR/Text $\rightarrow$ Лингвистический `Chunker` (разбиение на смысловые блоки).
*   [ ] **Экстрактор (LLM):** Внедрить предложенный промпт для извлечения сущностей и `claims` в формате JSON.
*   [ ] **Запись в граф:** Написать скрипт, который переводит JSON-выход LLM в `Cypher`-запросы для создания узлов и связей в Neo4j.

### 3. Интеллектуальный поиск (RAG и Векторы)
*   [ ] **Embeddings:** Настроить векторизацию `chunks` и `claims` (использование модели типа BGE или OpenAI embeddings).
*   [ ] **Векторный поиск:** Настроить базу (Qdrant/pgvector) для хранения векторов.
*   [ ] **Диалоговый агент:** Интегрировать промпт агента так, чтобы он в момент вопроса пользователя забирал из векгтора/графа `context` и отдавал ответ пользователю.

### 4. Фронтенд (UI)
*   [ ] **Загрузка:** Форма drag-and-drop для загрузки статей.
*   [ ] **Визуализация графа:** Отображение данных из Neo4j (используя Cytoscape.js).
*   [ ] **Чат-панель:** Окно диалога с агентом, поддерживающее уточняющие вопросы.

### 5. Резолюция сущностей (Базовый уровень)
